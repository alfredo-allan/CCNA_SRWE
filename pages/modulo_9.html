9.0.1 Por que devo cursar este módulo?

Sua rede está funcionando. Você conquistou a redundância da Camada 2 sem nenhum loop da Camada 2. Todos os seus dispositivos recebem seus endereços dinamicamente. Você é good na administração de rede! Mas, espere. Um de seus roteadores, o roteador de Default Gateway, na verdade, foi desativado. Nenhum de seus hosts pode enviar mensagens fora da rede imediata. Vai demorar um pouco para que este roteador de Default Gateway volte a funcionar. Você tem um monte de pessoas furiosas perguntando quando a emissora estará "de volta".

Você pode evitar esse problema facilmente. Os Protocolos de Redundância de First Hop (FHRPs) são a solução de que você precisa. Este módulo discute o que o FHRP faz e todos os tipos de FHRPs que estão disponíveis para você. Um desses tipos é um FHRP proprietário da Cisco chamado Hot Standby Router Protocol (HSRP). Você aprenderá como o HSRP funciona e, em seguida, concluirá uma atividade de Packet Tracer na qual você configurará e verificará o HSRP. Não espere, comece!

Dica: FHRPs são essenciais para garantir a continuidade dos serviços em redes corporativas, proporcionando failover transparente para os hosts finais.

9.1 Introdução ao FHRP

Os Protocolos de Redundância de First Hop (FHRPs) são mecanismos que fornecem alta disponibilidade para o gateway padrão em uma rede. Eles permitem que múltiplos roteadores compartilhem um único endereço IP virtual, garantindo que, se um roteador falhar, outro assuma imediatamente suas funções.

Tipos de FHRP:

  • HSRP (Hot Standby Router Protocol) - Protocolo proprietário Cisco
  • VRRP (Virtual Router Redundancy Protocol) - Padrão IETF (RFC 5798)
  • GLBP (Gateway Load Balancing Protocol) - Protocolo proprietário Cisco com balanceamento de carga
Sucesso: Com FHRP, os hosts continuam enviando tráfego para o gateway virtual, mesmo durante falhas de hardware ou manutenção programada.

9.1.1 Limitações do gateway padrão

Se um roteador ou uma interface de roteador (que serve como um Default Gateway) falhar, os hosts configurados com aquele Default Gateway são isolados das redes externas. É necessário um mecanismo para fornecer Default Gateways alternativos em redes comutadas nas quais dois ou mais roteadores são conectados às mesmas VLANs. Esse mecanismo é fornecido por Protocolos de Redundância de First Hop (FHRPs).

Problema Crítico: Em uma rede comutada, cada cliente recebe somente um Default Gateway. Não há como usar um gateway secundário, mesmo se um segundo caminho existir para levar os pacotes para fora do segmento local.

Os dispositivos finais geralmente são configurados com um único endereço IPv4 para um Default Gateway. Esse endereço não é alterado quando a topologia de rede é alterada. Se esse endereço IPv4 do Default Gateway não puder ser alcançado, o dispositivo local não poderá enviar pacotes do segmento de rede local, desconectando-o efetivamente de outras redes. Se existe um roteador redundante que possa servir como um Default Gateway para esse segmento, não existe método dinâmico pelo qual esses dispositivos possam determinar o endereço de um novo Default Gateway.

Observação: O IPv6 recebeu seu endereço de Default Gateway dinamicamente do Anúncio do Router ICMPv6. No entanto, os dispositivos IPv6 se beneficiam com um failover mais rápido para o novo Default Gateway ao usar o FHRP.
Impacto da Falha: Quando R1 falha, o PC1 (10.1.10.3/24) fica completamente isolado:
  • O PC1 continua enviando pacotes para R1 (10.1.10.2)
  • Os pacotes são descartados (Blackholing)
  • Não há failover automático para R2 ou R3
  • O servidor (10.9.1.50) fica inacessível
  • Usuários perdem conectividade com a rede externa

9.1.2 Redundância de roteador

Uma maneira de impedir um único ponto de falha no Default Gateway é implementar um roteador virtual. Para implementar esse tipo de redundância de roteador, vários roteadores são configurados para funcionar juntos, a fim de dar a ilusão de um único roteador para os hosts na LAN, como mostrado na figura. Ao compartilhar um endereço IP e um endereço MAC, dois ou mais roteadores podem atuar como um único roteador virtual.

O endereço IPv4 do roteador virtual é configurado como o Default Gateway para as estações de trabalho em um segmento IPv4 específico. Quando quadros são enviados de dispositivos host para o Default Gateway, os hosts usam ARP para resolver o endereço MAC associado ao endereço IPv4 do Default Gateway. A resolução ARP retorna o endereço MAC do roteador virtual. Os quadros enviados para o endereço MAC do roteador virtual podem, então, ser fisicamente processados pelo atual roteador ativo dentro do grupo do roteador virtual. Utiliza-se um protocolo para identificar dois ou mais roteadores como os dispositivos responsáveis pelo processamento dos quadros enviados para o MAC ou para o endereço IP de um único roteador virtual. Os dispositivos host enviam o tráfego para o endereço do roteador virtual. O roteador físico que encaminha esse tráfego é transparente para os dispositivos host.

Transparência: Um protocolo de redundância proporciona o mecanismo para determinar qual roteador deve assumir a função ativa no tráfego de encaminhamento. Ele também determina quando a função de encaminhamento deve ser assumida por um roteador em Standby. A transição de um roteador de transmissão para outro é transparente para os dispositivos finais.
Redundância de Primeiro Salto: A capacidade de uma rede de se recuperar dinamicamente da falha de um dispositivo que atua como um Default Gateway é conhecida como redundância de primeiro salto.

Como funciona a Redundância de Roteador?

1. Hosts usam Gateway Virtual (192.0.2.1) 2. ARP resolve MAC do Roteador Virtual 3. Roteador Ativo processa o tráfego
4. Falha no Roteador Ativo 5. Roteador Standby assume (failover) 6. Transparente para os hosts! ✅

9.1.4 Opções do FHRP

O FHRP usado em um ambiente de produção depende em grande parte do equipamento e das necessidades da rede. A tabela lista todas as opções disponíveis para FHRPs.

Opções do FHRP Descrição
HSRP
Proprietário Cisco
IPv4
 HSRP é um FHRP proprietário da Cisco que foi projetado para permitir failover transparente de um dispositivo IPv4 de primeiro salto. O HSRP fornece alta disponibilidade de rede fornecendo redundância de roteamento de primeiro salto para IPv4 hosts em redes configuradas com um endereço de Default Gateway IPv4. HSRP é usado em um grupo de roteadores para selecionar um dispositivo ativo e um dispositivo de Standby. Em um grupo de interfaces de dispositivo, o dispositivo ativo é o dispositivo usado para rotear pacotes; o dispositivo em Standby é o dispositivo que assume quando o dispositivo ativo falha ou quando condições pré-definidas são atendidas. A função do roteador de Standby HSRP é monitorar o status operacional do grupo HSRP e assumir rapidamente responsabilidade de encaminhamento de pacotes se o roteador ativo falhar.
HSRP para IPv6
Proprietário Cisco
IPv6
 Este é um FHRP proprietário da Cisco que fornece a mesma funcionalidade do HSRP, mas em um ambiente IPv6. Um grupo HSRP IPv6 tem um MAC virtual derivado do número do grupo HSRP e de um link IPv6 virtual derivado do endereço MAC virtual HSRP. Anúncios de Roteador (RAs) periódicos são enviados para o HSRP virtual IPv6 link local quando o grupo HSRP está ativo. Quando o grupo se torna inativo, esses RAs param depois que um RA final é enviado.
VRRPv2
Padrão Aberto (RFC 3768)
IPv4
 Este é um protocolo eleitoral não proprietário que atribui dinamicamente responsabilidade por um ou mais roteadores virtuais para os roteadores VRRP em uma LAN IPv4. Isso permite que vários roteadores em um link de multiaccesso usem o mesmo endereço IPv4 virtual. Um roteador VRRP está configurado para executar o VRRP em conjunto com um ou mais outros roteadores conectados a uma LAN. Em uma configuração VRRP, um roteador é eleito como o roteador mestre virtual, com os outros roteadores atuando como backups, caso o roteador mestre virtual venha a falhar.
VRRPv3
Padrão Aberto (RFC 5798)
IPv4 e IPv6
 Isso fornece a capacidade de suportar endereços IPv4 e IPv6. VRRPv3 funciona em ambientes de múltiplos fornecedores e é mais escalável que o VRRPv2.
GLBP
Proprietário Cisco
IPv4
 Este é um FHRP proprietário da Cisco que protege o tráfego de dados de uma falha no roteador ou circuito, como HSRP e VRRP, ao mesmo tempo em que permite o balanceamento de carga (também chamado de compartilhamento de carga) entre um grupo de roteadores redundantes.
GLBP para IPv6
Proprietário Cisco
IPv6
 Este é um FHRP proprietário da Cisco que fornece a mesma funcionalidade do GLBP, mas em um ambiente IPv6. O GLBP para IPv6 fornece failover automático do roteador para os hosts IPv6 configurados com um único Default Gateway em uma LAN. Vários roteadores de primeiro salto na LAN se combinam para oferecer um único roteador IPv6 de primeiro salto virtual ao compartilhar o encaminhamento de pacotes IPv6.
IRDP
Padrão Aberto (RFC 1256)
IPv4 (Legado)
 Especificado na RFC 1256, o IRDP é uma solução FHRP legada. O IRDP permite IPv4 hosts localizar roteadores que fornecem conectividade IPv4 a outras redes IP (não locais).

9.2.1 Visão geral do HSRP

A Cisco fornece HSRP e HSRP para IPv6 como uma forma de evitar a perda de acesso externo à rede se o roteador padrão falhar.

HSRP é um FHRP de propriedade da Cisco, projetado para permitir o failover transparente de um dispositivo IP de primeiro salto.

O HSRP garante alta disponibilidade de rede, fornecendo redundância de roteamento de primeiro salto para hosts IP em redes configuradas com um endereço de Default Gateway IP. O HSRP é usado em um grupo de roteadores para selecionar um dispositivo ativo e um dispositivo em Standby. Em um grupo de interfaces de dispositivo, o dispositivo ativo é aquele usado para rotear pacotes; o dispositivo em Standby é aquele que assume controle quando o dispositivo ativo falha ou quando as condições predefinidas são atendidas. A função de roteador em Standby do HSRP é monitorar o status operacional do grupo de HSRP e assumir rapidamente a responsabilidade de encaminhamento de pacotes se o roteador ativo falhar.

9.2.2 Prioridade e preempção de HSRP

A função dos roteadores ativos e de Standby é determinada durante o processo de eleição do HSRP. Por padrão, o roteador com o endereço IPv4 mais alto numericamente é escolhido como o roteador ativo. No entanto, é sempre melhor controlar como a rede funcionará em condições normais, em vez de deixar ao acaso.

Prioridade do HSRP

A prioridade do HSRP pode ser usada para determinar o roteador ativo. O roteador com a maior prioridade do HSRP será o roteador ativo. Por padrão, a prioridade do HSRP é 100. Se as prioridades forem iguais, o roteador com o endereço IPv4 mais alto numericamente será escolhido como o roteador ativo.

Comando: Para configurar um roteador para ser o roteador ativo, use o comando standby priority no modo de configuração de interface. O intervalo da prioridade do HSRP é de 0 a 255. 
R1(config-if)# standby 1 priority 150
R1(config-if)# end

Preempção do HSRP

Por padrão, após um roteador se tornar o roteador ativo, ele continuará ativo, mesmo que outro roteador com uma prioridade de HSRP maior fique on-line.

Para forçar um novo processo de eleição de HSRP a ocorrer quando um roteador de prioridade mais alta estiver on-line, a preempção deve ser habilitada usando o comando standby preempt no modo de configuração de interface.

Preempção é a capacidade de um roteador HSRP acionar o processo de reeleição. Com a preempção ativada, um roteador on-line com a maior prioridade de HSRP assumirá a função de roteador ativo. 
R1(config-if)# standby 1 preempt
R1(config-if)# end
Observação: A preempção permite que um roteador se torne o roteador ativo, apenas se ele tiver uma prioridade mais alta. Um roteador ativado para preempção, com prioridade igual, mas um endereço IPv4 mais alto, não irá fazer a preempção de um roteador ativo.

9.2.3 Estados e temporizadores de HSRP

Um roteador pode ser o roteador HSRP ativo, responsável por encaminhar o tráfego para o segmento, ou pode ser um roteador HSRP passivo em standby, pronto para assumir a função ativa se o roteador ativo falhar. Quando uma interface é configurada com o HSRP ou é ativada pela primeira vez com uma configuração atual do HSRP, o roteador envia e recebe os pacotes de aviso do HSRP para iniciar o processo de determinar qual estado ele assumirá no grupo do HSRP.

Intervalo padrão: 10 segundos

Tempo que um roteador espera sem receber Hello antes de declarar o roteador ativo como falho.

Relação Hello/Hold

Hold = 3 × Hello

O Hold Timer é geralmente 3 vezes o Hello Timer para permitir que um ou dois Hellos sejam perdidos sem declarar falha.

9.3.1 O que eu aprendi neste módulo?

Parabéns! Você concluiu o módulo sobre FHRP Concepts. Vamos revisar os principais conceitos que você aprendeu.

First Hop Redundancy Protocols (FHRP)

Se um roteador ou interface de roteador que serve como Default Gateway falhar, os hosts configurados com esse Default Gateway serão isolados de redes externas. O FHRP fornece Default Gateways alternativos em redes comutadas em que dois ou mais roteadores estão conectados às mesmas VLANs.

Uma forma de impedir um único ponto de falha no Default Gateway é implementar um roteador virtual. Com um roteador virtual, vários roteadores são configurados para trabalhar juntos para apresentar a ilusão de um único roteador aos hosts da LAN.

Hot Standby Router Protocol (HSRP)

O HSRP é um FHRP de propriedade da Cisco, projetado para permitir o failover transparente de um dispositivo IP de primeiro salto.

O HSRP é usado em um grupo de roteadores para selecionar um dispositivo ativo e um dispositivo em Standby. Em um grupo de interfaces de dispositivo, o dispositivo ativo é aquele usado para rotear pacotes; o dispositivo em Standby é aquele que assume controle quando o dispositivo ativo falha ou quando as condições predefinidas são atendidas.

Proprietário Cisco

Protocolo exclusivo da Cisco para redundância de gateway.

Prioridade

O roteador com a maior prioridade será o roteador ativo (padrão: 100).

Preempção

Capacidade de um roteador HSRP acionar o processo de reeleição.

Estados

Inicial, Aprendizado, Escuta, Fala, Standby e Ativo.

Resumo dos Estados do HSRP
Estado Descrição Resumida
INICIAL Interface configurada ou ativada
APRENDIZADO Aguardando descobrir o IP virtual
ESCUTA Conhece o IP, mas não é Ativo/Standby
FALA Envia Hello e participa da eleição
STANDBY Candidato a Ativo, pronto para assumir
ATIVO Encaminhando tráfego ativamente

Conceitos-Chave

Gateway Virtual

IP/MAC compartilhado pelos roteadores

Roteador Ativo

Encaminha o tráfego ativamente

Roteador Standby

Assume se o Ativo falhar

Failover

Transição transparente para hosts
