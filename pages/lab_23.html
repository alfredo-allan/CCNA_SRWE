☁️ Configurando DHCPv4 em Redes Corporativas

Aprenda a implementar um servidor DHCP centralizado, configurar retransmissão entre filiais e tornar seu roteador um cliente DHCP — tudo em um único laboratório prático.

0.0 Visão Geral do Laboratório

Imagine que você é o administrador de rede de uma empresa com três filiais:

  • Filial 1 (R1): Setor Administrativo
  • Filial 2 (R2): Matriz (onde fica o servidor DHCP)
  • Filial 3 (R3): Setor de Produção

Seu desafio é fazer com que todos os computadores recebam IP automaticamente, mesmo que o servidor DHCP esteja apenas na matriz. É como ter um "entregador de IPs" que viaja entre as filiais para entregar endereços!

Objetivos deste Lab:
  • Configurar o R2 como servidor DHCP para duas redes diferentes
  • Implementar retransmissão DHCP (ip helper-address) nos roteadores R1 e R3
  • Configurar o R2 como cliente DHCP na interface WAN
  • Validar a conectividade entre todos os dispositivos

2.1 Configurar R2 como Servidor DHCP

Passo 1: Excluir Endereços Reservados

🎓 Por que fazer isso? Para evitar que dois dispositivos tenham o mesmo IP — seria como ter duas casas com o mesmo número! Precisamos reservar os IPs dos gateways e de outros equipamentos estáticos.

Instrução de Validação: Os IPs .1 a .10 de cada rede DEVEM estar excluídos. O validador do Packet Tracer verifica essa condição.

Acesse o R2:

enable
configure terminal

Comando 1: Reservar IPs da rede 192.168.10.0/24 (Filial Administrativa)

R2(config)# ip dhcp excluded-address 192.168.10.1 192.168.10.10
  • ip dhcp excluded-address → Diz ao DHCP quais IPs NÃO podem ser entregues
  • 192.168.10.1 192.168.10.10 → Bloqueia do IP .1 até .10
  • .1 → É o gateway do R1 (interface G0/0)
  • .2 a .10 → Reservados para servidores e equipamentos importantes

Comando 2: Reservar IPs da rede 192.168.30.0/24 (Filial de Produção)

R2(config)# ip dhcp excluded-address 192.168.30.1 192.168.30.10
Analogia do Dia a Dia: É como reservar os 10 primeiros lugares de um estacionamento para os diretores da empresa. Ninguém mais pode usar esses lugares!

Passo 2: Pool DHCP para LAN do R1 (R1-LAN)

🎓 O que vamos fazer? Criar uma "caixa" de endereços para a Filial Administrativa. Cada computador dessa filial vai pegar um IP dessa caixa. 

R2(config)# ip dhcp pool R1-LAN
R2(dhcp-config)# network 192.168.10.0 255.255.255.0
R2(dhcp-config)# default-router 192.168.10.1
R2(dhcp-config)# dns-server 192.168.20.254
R2(dhcp-config)# exit
  • ip dhcp pool R1-LAN → Cria a "caixa" com nome R1-LAN
  • network 192.168.10.0 255.255.255.0 → Define a rede disponível (de .1 até .254)
  • default-router 192.168.10.1 → Define o "porteiro" que direciona o tráfego para fora
  • dns-server 192.168.20.254 → Define o servidor que traduz nomes em IPs
Analogia do Dia a Dia: O DNS é como uma "lista telefônica" da internet — você fala o nome da pessoa e ela te dá o número!

Passo 3: Pool DHCP para LAN do R3 (R3-LAN)

🎓 O que vamos fazer? Criar uma "caixa" de endereços para a Filial de Produção.

R2(config)# ip dhcp pool R3-LAN
R2(dhcp-config)# network 192.168.30.0 255.255.255.0
R2(dhcp-config)# default-router 192.168.30.1
R2(dhcp-config)# dns-server 192.168.20.254
R2(dhcp-config)# exit

2.2 Configurar Retransmissão DHCP

Entendendo a Retransmissão

🎓 Explicação Humanizada: Agora temos um problema: o servidor DHCP (R2) está na matriz, mas os computadores das filiais R1 e R3 estão em redes diferentes. Como eles vão pedir IP se estão "longe" do servidor?

É aí que entra o "agente de retransmissão" (ip helper-address). Pense como um "carteiro especial" que fica nas filiais:

  • 📤 Quando um PC pede um IP, o carteiro pega o pedido
  • 📦 Ele entrega para o servidor DHCP na matriz
  • 📨 O servidor devolve a resposta com o IP
  • 📬 O carteiro entrega o IP para o PC

Sem esse "carteiro", os pedidos de IP nunca chegariam ao servidor!

Passo 1: Configurar R1 como Agente de Retransmissão

Acesse o R1:

enable
configure terminal
R1(config)# interface g0/0
R1(config-if)# ip helper-address 10.1.1.2
  • ip helper-address → Ativa o "carteiro" na interface
  • 10.1.1.2 → Endereço do servidor DHCP (interface S0/0/0 do R2)
  • Todo pedido DHCP vindo da LAN será encaminhado para esse IP
Analogia: É como ter um porteiro no prédio que recebe as encomendas e avisa o morador. O porteiro é o "helper-address"!

Passo 2: Configurar R3 como Agente de Retransmissão

Acesse o R3:

enable
configure terminal
R3(config)# interface g0/0
R3(config-if)# ip helper-address 10.2.2.2
ATENÇÃO! Note que o IP é diferente!
  • R1 usa 10.1.1.2 (conexão serial entre R1 e R2)
  • R3 usa 10.2.2.2 (conexão serial entre R3 e R2)

Passo 3: Configurar PCs para DHCP

No PC1 (Filial Administrativa):

  1. Clique no PC1 → Desktop → IP Configuration
  2. Selecione DHCP
  3. Aguarde alguns segundos

No PC2 (Filial de Produção):

  1. Clique no PC2 → Desktop → IP Configuration
  2. Selecione DHCP
  3. Aguarde alguns segundos
O que está acontecendo? Os PCs estão enviando um pedido de IP para toda a rede. O R1/R3 (carteiro) captura esse pedido e encaminha para o R2. O R2 (servidor) responde com um IP da pool correta. O PC recebe o IP e pode se comunicar!

3.1 Configurar R2 como Cliente DHCP

Configurar Interface G0/1 do R2

🎓 Explicação Humanizada: Agora vamos fazer algo interessante: o próprio roteador R2 vai pedir um IP para a internet!

Imagine que sua empresa contrata um provedor de internet. O provedor te dá um cabo e diz: "conecte aqui e você vai receber um IP automaticamente". Isso é exatamente o que vamos fazer com a interface G0/1 do R2.

Por que isso é útil?

  • ✅ O provedor de internet pode mudar seu IP a qualquer momento
  • ✅ Se você configurar manualmente, teria que alterar toda vez
  • ✅ Com DHCP, o roteador se adapta automaticamente!

No R2:

configure terminal
R2(config)# interface g0/1
R2(config-if)# ip address dhcp
R2(config-if)# no shutdown
R2(config-if)# exit
  • interface g0/1 → Entra na interface que vai para o ISP
  • ip address dhcp → Diz "me dê um IP automaticamente"
  • no shutdown → Liga a interface (estava desligada)
DICA IMPORTANTE: No Packet Tracer, após digitar os comandos, clique no botão "Fast Forward Time" (canto inferior direito) para acelerar a obtenção do IP!

Verificar se recebeu o IP:

R2# show ip interface brief
  • 🔍 A interface G0/1 deve ter um IP (ex: 209.165.200.226)
  • 🔍 O status deve ser up/up
  • 🔍 Se aparecer "unassigned", significa que não recebeu IP
Analogia: É como chegar em um hotel e receber um quarto automaticamente na recepção. Você não precisa escolher o número, eles te dão um disponível!

4.1 Verificar DHCP e Conectividade

Verificar Associações DHCP

No R2:

R2# show ip dhcp binding

Saída esperada:

IP address      Client-ID/          Lease expiration    Type
                Hardware address
192.168.10.11   0002.4AA5.1470      --                  Automatic
192.168.30.11   0004.9A97.2535      --                  Automatic
  • Coluna 1: IP entregue ao PC
  • Coluna 2: Endereço MAC do PC (identificador único)
  • Coluna 3: Tempo de validade (-- significa permanente no lab)
  • Coluna 4: Tipo de atribuição (automática)
Como saber qual é qual?
  • PC1 (MAC: 0002.4AA5.1470) recebeu 192.168.10.11
  • PC2 (MAC: 0004.9A97.2535) recebeu 192.168.30.11

📖 Analogia: É como olhar uma lista de hóspedes do hotel: "Quarto 101 - João Silva, Quarto 102 - Maria Santos"

Testar Conectividade

Teste 1: PC1 para PC2

PC1> ping 192.168.30.11

Teste 2: PC1 para DNS Server

PC1> ping 192.168.20.254

Teste 3: PC2 para R1

PC2> ping 192.168.10.1

Teste 4: PC1 para R3

PC1> ping 192.168.30.1
O que esperar? Todos os pings devem ser bem-sucedidos (resposta com 0% de perda). Isso confirma que a rede está funcionando perfeitamente!
Analogia: É como fazer uma chamada de teste entre todos os departamentos da empresa para garantir que a comunicação está funcionando.

5.1 Comandos de Diagnóstico

Use estes comandos para verificar e diagnosticar sua configuração:

Comando O que faz? Quando usar?
show ip dhcp binding Lista todos os IPs entregues Para ver quem recebeu IP
show ip dhcp pool Mostra as pools configuradas Para ver as "caixas" de IPs
show ip interface brief Mostra IPs das interfaces Para verificar IPs dos roteadores
show running-config | section dhcp Mostra configurações DHCP Para revisar o que foi configurado
show ip route Mostra tabela de roteamento Para verificar caminhos na rede

6.1 Erros Comuns e Soluções

Problema Causa Provável Solução
PC não recebe IP Helper-address não configurado Verificar R1 e R3: show running-config interface g0/0
R2 não recebe IP no G0/1 Interface desligada Verificar no shutdown
IP duplicado Endereço não excluído Verificar show ip dhcp excluded-address
Ping não funciona Roteamento ausente Configurar rotas estáticas ou protocolo de roteamento
Pool DHCP não funciona Gateway errado Verificar default-router na pool

Use esta lista para garantir que sua configuração está 100% correta:
