Configuração e Endurecimento do SSH

Substituição do protocolo Telnet por conexões criptografadas seguras e correção do bug de usuário depreciado.

2.1 Tabela de Endereçamento do Escopo

Os seguintes parâmetros de rede são utilizados para validar o acesso seguro:

Dispositivo Interface Endereço IP Máscara de Sub-rede
S1 VLAN 1 10.10.10.2 255.255.255.0
PC1 NIC 10.10.10.10 255.255.255.0

2.2 Parte 1: Criptografia Local de Senhas

Primeiro acesso via Telnet para criptografar as senhas globais que estavam armazenadas em texto claro:

S1> enable
Senha: cisco
S1# configure terminal

! Ativando criptografia do arquivo running-config
S1(config)# service password-encryption
S1(config)# exit

2.3 Parte 2: Geração de Chaves RSA e Configuração de VTY

Configuração do domínio, geração do par de chaves assimétricas e restrição total das linhas VTY para aceitarem apenas SSH:

Aviso de Correção (100% de Conclusão): Não utilize a palavra-chave password ao criar o usuário local. Os validadores modernos exigem o comando secret para evitar o erro de usuário depreciado (deprecated username). 
S1# configure terminal

! Configurando domínio e chaves RSA de 1024 bits
S1(config)# ip domain-name netacad.pka
S1(config)# crypto key generate rsa
! Quando solicitado o tamanho, digite: 1024

! CORREÇÃO DO USERNAME: Utilizando o comando secret (obrigatório)
S1(config)# username administrator secret cisco

! Configurando as linhas Virtuais (VTY)
S1(config)# line vty 0 15
S1(config-line)# no password
S1(config-line)# login local
S1(config-line)# transport input ssh
S1(config-line)# exit
S1# copy running-config startup-config

2.4 Parte 3: Validação e Testes de Conectividade

Para confirmar que a segurança foi implementada corretamente, execute os testes no prompt do PC1:

1. Teste de bloqueio do Telnet (Deve falhar):

C:\> telnet 10.10.10.2
Trying 10.10.10.2 ...
% Connection timed out; remote host not responding

2. Teste de conexão SSH segura (Deve funcionar):

C:\> ssh -l administrator 10.10.10.2
Open
Password: cisco

S1>
Conexão estabelecida com sucesso usando criptografia simétrica e assimétrica de ponta a ponta!
