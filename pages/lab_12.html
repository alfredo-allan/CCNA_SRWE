3.0 Roteiro de Execução na CLI (Comandos Comentados)

3.1 Configuração Avançada no Switch S1

Aplica os parâmetros globais de segurança, provisiona as sub-redes virtuais, isola as interfaces não utilizadas na VLAN de descarte e amarra o Tronco com poda de tráfego. 

S1> enable
! Concede acesso ao modo EXEC Privilegiado de administração.

S1# configure terminal
! Transiciona o terminal para o modo de configuração global.

S1(config)# hostname S1
! Define a identidade do switch na topologia.

S1(config)# no ip domain-lookup
! Previne que o switch tente traduzir comandos errados via DNS, evitando o travamento da CLI.

S1(config)# enable secret class
! Aplica encriptação forte do tipo hash para proteger a entrada no modo enable.

S1(config)# line console 0
S1(config-line)# password cisco
S1(config-line)# login
S1(config-line)# exit
! Blinda a porta física de console local exigindo autenticação.

S1(config)# line vty 0 15
S1(config-line)# password cisco
S1(config-line)# login
S1(config-line)# exit
! Ativa e protege com senha as 16 linhas lógicas para conexões de rede remotas.

S1(config)# service password-encryption
! Transforma as senhas armazenadas de texto simples para formato criptografado no arquivo de configuração.

S1(config)# banner motd "ACESSO NAO AUTORIZADO E PROIBIDO!"
! Exibe um aviso de segurança jurídico restritivo no momento do logon.

! --- BANCO DE DADOS DE VLANS (CRIÇÃO) ---
S1(config)# vlan 10
S1(config-vlan)# name Gerenciamento
S1(config-vlan)# vlan 20
S1(config-vlan)# name Vendas
S1(config-vlan)# vlan 30
S1(config-vlan)# name Operacoes
S1(config-vlan)# vlan 999
S1(config-vlan)# name ParkingLot
S1(config-vlan)# vlan 1000
S1(config-vlan)# name Nativa
S1(config-vlan)# exit
! Cria as tabelas de ID e associa os nomes exatos do gabarito.

! --- CONFIGURAÇÃO DE SVIs (INTERFACES VIRTUAIS) ---
S1(config)# interface vlan 10
S1(config-if)# ip address 192.168.10.11 255.255.255.0
S1(config-if)# no shutdown

S1(config)# interface vlan 20
S1(config-if)# ip address 192.168.20.11 255.255.255.0
S1(config-if)# no shutdown

S1(config)# interface vlan 30
S1(config-if)# ip address 192.168.30.11 255.255.255.0
S1(config-if)# no shutdown
S1(config-if)# exit
! Configura e ativa as três instâncias IPs administrativas solicitadas para o S1.

! --- MAPEAMENTO DE PORTAS DE ACESSO ---
S1(config)# interface FastEthernet0/6
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 20
S1(config-if)# exit
! Fixa a interface do PC-A no modo de acesso atrelado à rede de Vendas (VLAN 20).

! --- MITIGAÇÃO DE SEGURANÇA (PARKING LOT) ---
S1(config)# interface range FastEthernet0/2 - 5, FastEthernet0/7 - 24, GigabitEthernet0/1 - 2
S1(config-if-range)# switchport mode access
S1(config-if-range)# switchport access vlan 999
S1(config-if-range)# shutdown
S1(config-if-range)# exit
! Agruda todas as 22 portas inativas do S1, move para o "buraco negro" da VLAN 999 e desliga a energia delas.

! --- ESTRUTURAÇÃO DO TRONCO 802.1Q SEGURO ---
S1(config)# interface FastEthernet0/1
S1(config-if)# switchport mode trunk
! Força o cabo de uplink com o S2 a operar estritamente como Trunking fixo.

S1(config-if)# switchport trunk native vlan 1000
! Modifica a VLAN nativa para a VLAN 1000 isolada, impedindo ataques de injeção de pacotes.

S1(config-if)# switchport trunk allowed vlan 10,20,30,1000
! Poda de tráfego: Bloqueia a transmissão de qualquer outra VLAN (incluindo a 999 e a 1) pelo tronco.
S1(config-if)# exit
S1(config)# end

S1# write memory
! Persiste os dados lógicos na memória não-volátil (NVRAM).

3.2 Configuração Avançada no Switch S2

Aplica as mesmas diretivas globais de segurança, associa a interface de acesso do PC-B e replica a higienização de portas e configurações do tronco. 

S2> enable
S2# configure terminal

S2(config)# hostname S2
S2(config)# no ip domain-lookup
S2(config)# enable secret class

! [Linhas de console, vty, encriptação de senhas e banner aplicados de forma idêntica ao S1]

! --- BANCO DE DADOS DE VLANS (CRIÇÃO) ---
S2(config)# vlan 10
S2(config-vlan)# name Gerenciamento
S2(config-vlan)# vlan 20
S2(config-vlan)# name Vendas
S2(config-vlan)# name Operacoes
S2(config-vlan)# vlan 999
S2(config-vlan)# name ParkingLot
S2(config-vlan)# vlan 1000
S2(config-vlan)# name Nativa
S2(config-vlan)# exit

! --- CONFIGURAÇÃO DE SVI DE GERÊNCIA NO S2 ---
S2(config)# interface vlan 10
S2(config-if)# ip address 192.168.10.12 255.255.255.0
S2(config-if)# no shutdown
S2(config-if)# exit

! --- MAPEAMENTO DE PORTA DE ACESSO ---
S2(config)# interface FastEthernet0/18
S2(config-if)# switchport mode access
S2(config-if)# switchport access vlan 30
S2(config-if)# exit
! Vincula a interface do PC-B na rede de Operações (VLAN 30).

! --- MITIGAÇÃO DE SEGURANÇA (PARKING LOT) ---
S2(config)# interface range FastEthernet0/2 - 17, FastEthernet0/19 - 24, GigabitEthernet0/1 - 2
S2(config-if-range)# switchport mode access
S2(config-if-range)# switchport access vlan 999
S2(config-if-range)# shutdown
S2(config-if-range)# exit
! Isola e desliga administrativamente as 22 portas ociosas do switch S2.

! --- ESTRUTURAÇÃO DO TRONCO 802.1Q SEGURO ---
S2(config)# interface FastEthernet0/1
S2(config-if)# switchport mode trunk
S2(config-if)# switchport trunk native vlan 1000
S2(config-if)# switchport trunk allowed vlan 10,20,30,1000
S2(config-if)# exit
S2(config)# end

S2# write memory

4.0 Glossário de Conceitos de Segurança de Redes

Entende a terminologia avançada de engenharia utilizada para mitigar ataques e estruturar a arquitetura lógica deste cenário:

Termo de Engenharia Analogia Didática e Significado Prático
Parking Lot (VLAN de Mitigação) Um "quarto escuro blindado". É uma VLAN isolada criada unicamente para prender as portas do switch que não estão a ser usadas, garantindo que se alguém plugar um cabo nelas por maldade, não terá acesso a nenhum dado da empresa.
Poda de Tronco (Allowed VLAN List) Um "filtro de portaria". Por padrão, um cabo de tronco deixa passar dados de todas as redes. A lista de permissões atua limitando o cabo a carregar estritamente as VLANs autorizadas, descartando o resto para economizar banda e segurança.
VLAN Nativa Estrita (VLAN 1000) Um canal de descarte de frames sem etiqueta. Mudar a VLAN nativa da padrão (VLAN 1) para um ID exclusivo (VLAN 1000) serve para neutralizar técnicas de invasão onde o hacker injeta etiquetas falsas para pular de rede.
Shutdown Administrativo Cortar a eletricidade da tomada. Desativar o comando de energia impede que loops acidentais aconteçam e barra intrusões físicas nas tomadas de rede espalhadas pelo prédio.
SVI Múltipla Criar portas de administração independentes para sub-redes isoladas, permitindo que a equipa de suporte consiga gerenciar o ativo de diferentes pontos de acesso lógicos.

5.0 Casos de Uso no Ambiente Corporativo (Humanizado)

Caso 1: Proteção Física de Escritórios contra Intrusos (Parking Lot & Shutdown)

Imagina uma empresa onde existem tomadas de rede vazias nas paredes das salas de reunião e corredores públicos. Se um visitante mal-intencionado ou um invasor plugar o seu próprio portátil numa dessas tomadas, ele estaria imediatamente dentro da rede interna da empresa se as portas estivessem ativas. Ao mover todas as portas vazias para o ParkingLot (VLAN 999) e aplicar o comando shutdown, essas tomadas tornam-se "mortas". O invasor não recebe sinal de rede e fica completamente isolado, sem chance de capturar informações confidenciais.

Caso 2: Otimização de Performance e Controle de Tráfego (Allowed VLAN List)

Numa grande organização com centenas de computadores, existem tráfegos automáticos pesados que as máquinas enviam constantemente para descobrir impressoras e servidores (chamados broadcasts). Se o cabo de tronco que interliga os switches não tiver restrições, esse "ruído de fundo" de todas as redes viaja por toda a infraestrutura desnecessariamente, deixando os links lentos. Ao aplicar a poda de tronco (allowed vlan), nós definimos regras claras: pelo cabo principal só passa o que for estritamente vital. O ruído das portas isoladas do estoque ou de testes fica bloqueado, otimizando a velocidade da internet e dos sistemas.
