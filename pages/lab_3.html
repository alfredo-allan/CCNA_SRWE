3.2 Parte 1: Ativação e Endereçamento IPv4 no R1 [cite: 322]

Configuração das sub-redes das interfaces LAN do roteador R1. Lembre-se de utilizar a máscara de sub-rede correspondente ao prefixo /25 (255.255.255.128)[cite: 320, 331]:

Credenciais do R1: Senha de usuário EXEC: cisco | Senha EXEC Privilegiado: class[cite: 327]. 
R1> enable
Senha: class
R1# configure terminal

! Configurando e subindo a Interface GigabitEthernet0/0
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip address 172.16.20.1 255.255.255.128
R1(config-if)# no shutdown
R1(config-if)# exit

! Configurando e subindo a Interface GigabitEthernet0/1
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip address 172.16.20.129 255.255.255.128
R1(config-if)# no shutdown
R1(config-if)# exit
R1# copy running-config startup-config

3.3 Parte 2: Configuração IPv6 Global e Link-Local no R2 [cite: 323, 340]

Ativação do encaminhamento de pacotes IPv6 global (mecanismo essencial) e definição dos escopos nas portas do roteador R2[cite: 340]:

Atenção Obrigatória: Sem o comando ipv6 unicast-routing no modo global, o roteador não processará as mensagens de vizinhança e os pings de teste entre os computadores falharão. 
R2> enable
Senha: class
R2# configure terminal

! ATIVAÇÃO GLOBAL DO ENCAMINHAMENTO IPV6 (Obrigatório)
R2(config)# ipv6 unicast-routing

! Configurando Interface GigabitEthernet0/0
R2(config)# interface GigabitEthernet0/0
R2(config-if)# ipv6 address 2001:db8:c0de:12::1/64
R2(config-if)# ipv6 address fe80::2 link-local
R2(config-if)# no shutdown
R2(config-if)# exit

! Configurando Interface GigabitEthernet0/1
R2(config)# interface GigabitEthernet0/1
R2(config-if)# ipv6 address 2001:db8:c0de:13::1/64
R2(config-if)# ipv6 address fe80::2 link-local
R2(config-if)# no shutdown
R2(config-if)# exit
R2# copy running-config startup-config

3.4 Validação Prática de Conectividade [cite: 333, 343]

Após definir estaticamente as configurações lógicas nas placas de rede (NIC) de todos os computadores, abra os prompts de comando para testar a comunicação inter-redes[cite: 331, 334, 341]:

No Prompt do PC1 (Teste IPv4 para PC2)[cite: 334]:

C:\> ping 172.16.20.138
Pinging 172.16.20.138 with 32 bytes of data:
Reply from 172.16.20.138: bytes=32 time<1ms TTL=127
Packets: Sent = 4, Received = 4, Lost = 0 (0% loss)

No Prompt do PC3 (Teste IPv6 para PC4)[cite: 344]:

C:\> ping 2001:db8:c0de:13::a
Pinging 2001:db8:c0de:13::a with 32 bytes of data:
Reply from 2001:db8:c0de:13::a: bytes=32 time<1ms TTL=127
Packets: Sent = 4, Received = 4, Lost = 0 (0% loss)
Interfaces ativas e protocolo Dual-Stack operando perfeitamente. 100% de conclusão validado com sucesso!
