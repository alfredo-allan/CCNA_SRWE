2.0 Resolução de Problemas no R1 (IPv4)

Tratamento da interface local que se encontra desativada administrativamente e correção imediata do desalinhamento de máscara /30 detectado na interface Serial.

Instrução de Validação: O IP inicial 209.165.200.229 que constava na CLI pertence a outra subrede. Ao aplicar o IP correto 209.165.200.225 com a máscara correta (255.255.255.252), o Packet Tracer computará a pontuação devida. 
R1> enable
Password: class
R1# configure terminal

! Ativando a interface LAN que estava em administratively down
R1(config)# interface GigabitEthernet0/0/0
R1(config-if)# no shutdown

! Garantindo a configuração da segunda LAN
R1(config)# interface GigabitEthernet0/0/1
R1(config-if)# ip address 172.16.20.129 255.255.255.128
R1(config-if)# no shutdown

! CORREÇÃO DO IP/MÁSCARA DA INTERFACE SERIAL
R1(config)# interface Serial0/1/0
R1(config-if)# ip address 209.165.200.225 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# end
R1# write memory

3.0 Resolução de Problemas no R2 (IPv6)

Como as interfaces IPv6 aceitam múltiplos endereços simultâneos, para sanar o erro de digitação do cenário, deve-se remover a rede inválida explicitamente para limpar a tabela de vizinhança.

Atenção ao Bug do Simulador: Caso esqueça de digitar o comando no ipv6 address, a interface manterá ambos os IPs ativos, impedindo a validação completa do laboratório em 100%. 
R2> enable
Password: class
R2# show ipv6 interface brief
R2# configure terminal
R2(config)# interface GigabitEthernet0/0/1
R2(config-if)# no ipv6 address 2001:db8:c0de:14::1/64
R2(config-if)# ipv6 address 2001:db8:c0de:13::1/64
R2(config-if)# no shutdown
R2(config-if)# end
R2# write memory

4.0 Verificação de Conectividade Finais
