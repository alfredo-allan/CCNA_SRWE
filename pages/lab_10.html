Laboratório 11

Configuração Avançada de VTP e DTP

Centralização de banco de dados de VLANs, automação de entroncamento e hardening de portas de infraestrutura de rede corporativa.

1.0 Roteiro Prático de Execução (CLI Oficial)

1.1 Switch Central S1 (VTP Server)

Configuração do tronco dinâmico para o S2, tronco estático sem negociação para o S3 e criação da base de dados de VLANs:

S1> enable
S1# configure terminal

! Configurando o protocolo DTP nas interfaces de Uplink
S1(config)# interface GigabitEthernet0/1
S1(config-if)# switchport mode dynamic desirable
S1(config-if)# exit

S1(config)# interface GigabitEthernet0/2
S1(config-if)# switchport mode trunk
S1(config-if)# switchport nonegotiate
S1(config-if)# exit

! Ativando e blindando o servidor VTP
S1(config)# vtp mode server
S1(config)# vtp domain CCNA
S1(config)# vtp password cisco

! Configurando o IP de Gerência na SVI
S1(config)# interface vlan 99
S1(config-if)# ip address 192.168.99.1 255.255.255.0
S1(config-if)# no shutdown
S1(config-if)# end
S1# write memory

1.2 Switch S2 (VTP Client)

Configuração do modo cliente e divisão em massa das portas de acesso nos intervalos corretos do laboratório:

S2> enable
S2# configure terminal

! Sincronizando o protocolo VTP no modo Cliente
S2(config)# vtp mode client
S2(config)# vtp domain CCNA
S2(config)# vtp password cisco

! Atribuição em massa de portas de acesso conforme o manual técnico
S2(config)# interface range fastEthernet 0/1 - 8
S2(config-if-range)# switchport mode access
S2(config-if-range)# switchport access vlan 10

S2(config)# interface range fastEthernet 0/9 - 16
S2(config-if-range)# switchport mode access
S2(config-if-range)# switchport access vlan 20

S2(config)# interface range fastEthernet 0/17 - 24
S2(config-if-range)# switchport mode access
S2(config-if-range)# switchport access vlan 30
S2(config-if-range)# end
S2# write memory

1.3 Switch S3 (VTP Client Estrito)

Desativação completa da negociação DTP na interface de uplink e mapeamento idêntico de portas de acesso:

S3> enable
S3# configure terminal

S3(config)# vtp mode client
S3(config)# vtp domain CCNA
S3(config)# vtp password cisco

! Desativando o DTP na porta G0/2 para casar com o S1
S3(config)# interface GigabitEthernet0/2
S3(config-if)# switchport mode trunk
S3(config-if)# switchport nonegotiate

! Mapeamento das portas FastEthernet do S3
S3(config)# interface range fastEthernet 0/1 - 8
S3(config-if-range)# switchport mode access
S3(config-if-range)# switchport access vlan 10

S3(config)# interface range fastEthernet 0/9 - 16
S3(config-if-range)# switchport mode access
S3(config-if-range)# switchport access vlan 20

S3(config)# interface range fastEthernet 0/17 - 24
S3(config-if-range)# switchport mode access
S3(config-if-range)# switchport access vlan 30
S3(config-if-range)# end
S3# write memory

2.0 Glossário de Engenharia de Redes

Entenda os conceitos e protocolos avançados empregados para estabilizar e validar esta topologia:

Termo Técnico Definição e Significado no Cenário Cisco
VTP (VLAN Trunking Protocol) Protocolo proprietário da Cisco de Camada 2 que automatiza a criação, exclusão e sincronização de VLANs através de uma arquitetura baseada em Servidores e Clientes.
VTP Domain Delimitação lógica de gerenciamento. Switches só trocam anúncios e atualizações de VLANs se compartilharem estritamente o mesmo nome de domínio e senha.
DTP (Dynamic Trunking Protocol) Protocolo dinâmico encarregado de negociar automaticamente o estado de uma interface, definindo se ela operará como uma porta de acesso ou um link de tronco.
Dynamic Desirable Estado operacional do DTP onde a interface ativamente envia pacotes para tentar converter o link elétrico vizinho em um tronco estruturado (Trunk).
Switchport Nonegotiate Mecanismo de segurança que desativa completamente o envio de frames DTP na interface, travando o Trunk de forma estática e economizando processamento e banda.
SVI (Switched Virtual Interface) Interface lógica (virtual) de Camada 3 configurada dentro de um switch de Camada 2 para atribuir um IP administrativo, viabilizando monitoramento remoto e SSH.

3.0 Aplicação no Mercado de Trabalho

Caso de Uso 1: Expansão de Campi e Prédios Corporativos (VTP)

Em uma empresa com múltiplos switches distribuídos por andares, criar uma nova VLAN para um departamento (ex: VLAN de Auditoria) exigiria acessar switch por switch de forma manual. Com o VTP configurado, o engenheiro de redes cria a VLAN apenas no Switch Core (Server). Em menos de um segundo, a alteração é propagada de forma limpa para toda a infraestrutura, eliminando o erro humano e o retrabalho.

Caso de Uso 2: Hardening de Infraestrutura contra VLAN Hopping (Nonegotiate)

Portas configuradas com negociação dinâmica padrão (DTP) podem ser exploradas por atacantes locais. Se um invasor conectar um computador modificado e simular frames de tronco, ele pode induzir o switch a abrir um Trunk, ganhando acesso ao tráfego de dados de todas as sub-redes da empresa. O uso do comando switchport nonegotiate mata a sinalização do DTP, blindando os uplinks contra interceptações maliciosas.
