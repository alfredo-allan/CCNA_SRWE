1.1 Configurações Básicas dos Dispositivos

Inicialização dos parâmetros globais de gerência, criptografia local e desativação de pesquisas DNS nos switches S1 e S2.

Instrução de Validação: O comando service password-encryption garante a conversão de todas as senhas em texto claro para hash criptográfico no arquivo de configuração ativa.

Procedimento de Linha de Comando (Aplicado em S1 e S2):

! Configurações globais em S1
S1(config)# hostname S1
S1(config)# no ip domain-lookup
S1(config)# enable secret class
S1(config)# line console 0
S1(config-line)# password cisco
S1(config-line)# login
S1(config-line)# exit
S1(config)# line vty 0 4
S1(config-line)# password cisco
S1(config-line)# login
S1(config-line)# exit
S1(config)# service password-encryption
S1(config)# banner motd # ACESSO NAO AUTORIZADO PROIBIDO #

! Repetir o mesmo padrão alterando apenas o hostname para S2

2.1 Criação de VLANs e Atribuição de Portas

Configuração das VLANs de dados, gerência e isolamento de segurança para contenção de tráfego broadcast nas portas inativas.

Aviso de Validação Técnica: Mover as interfaces não utilizadas para a VLAN 999 (Parking_Lot) e aplicar o comando shutdown é mandatório para alcançar os 100% no validador.

Configuração das Interfaces - Switch S1:

S1(config)# vlan 10
S1(config-vlan)# name Gerenciamento
S1(config-vlan)# vlan 20
S1(config-vlan)# name Clientes
S1(config-vlan)# vlan 999
S1(config-vlan)# name Parking_Lot
S1(config-vlan)# vlan 1000
S1(config-vlan)# name Nativa
S1(config-vlan)# exit
S1(config)# interface vlan 10
S1(config-if)# ip address 192.168.10.11 255.255.255.0
S1(config-if)# no shutdown
S1(config-if)# exit
S1(config)# interface f0/6
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 20
S1(config-if)# exit
S1(config)# interface range f0/3 - 5, f0/7 - 24, g0/1 - 2
S1(config-if-range)# switchport mode access
S1(config-if-range)# switchport access vlan 999
S1(config-if-range)# shutdown

Configuração das Interfaces - Switch S2:

S2(config)# vlan 10
S2(config-vlan)# name Gerenciamento
S2(config-vlan)# vlan 20
S2(config-vlan)# name Clientes
S2(config-vlan)# vlan 999
S2(config-vlan)# name Parking_Lot
S2(config-vlan)# vlan 1000
S2(config-vlan)# name Nativa
S2(config-vlan)# exit
S2(config)# interface vlan 10
S2(config-if)# ip address 192.168.10.12 255.255.255.0
S2(config-if)# no shutdown
S2(config-if)# exit
S2(config)# interface f0/18
S2(config-if)# switchport mode access
S2(config-if)# switchport access vlan 20
S2(config-if)# exit
S2(config)# interface range f0/3 - 17, f0/19 - 24, g0/1 - 2
S2(config-if-range)# switchport mode access
S2(config-if-range)# switchport access vlan 999
S2(config-if-range)# shutdown

3.1 Configuração de Troncos 802.1Q Estáticos

Forçamento manual do entroncamento nos links físicos de interconexão. Modificação da VLAN nativa para evitar ataques de salto de VLAN e definição de restrição de tráfego.

Comandos Executados nos Links de Interconexão (S1 e S2):

! Comando aplicado no range de portas de ambos os switches
S1(config)# interface range f0/1 - 2
S1(config-if-range)# switchport mode trunk
S1(config-if-range)# switchport trunk native vlan 1000
S1(config-if-range)# switchport trunk allowed vlan 10,20,1000

4.1 Configuração do EtherChannel Baseado em LACP

Agrupamento das portas físicas no canal lógico dinâmico utilizando a negociação ativa do protocolo padrão aberto IEEE 802.3ad.

Ponto Crítico de Validação: Para que a interface Port-Channel funcione de forma robusta e seja pontuada, ela precisa espelhar exatamente a configuração de trunking realizada nas portas físicas.

Configuração de Agrupamento Dinâmico (S1 e S2):

! Configuração do canal e amarração lógica
S1(config)# interface range f0/1 - 2
S1(config-if-range)# channel-group 1 mode active
S1(config-if-range)# exit
S1(config)# interface port-channel 1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport trunk native vlan 1000
S1(config-if)# switchport trunk allowed vlan 10,20,1000
