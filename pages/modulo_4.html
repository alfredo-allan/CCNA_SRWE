Roteamento Inter-VLANs

4.0.1 Por que devo cursar este módulo?

Bem-vindo ao roteamento entre VLANs!

Agora você sabe como segmentar e organizar sua rede em VLANs. Os hosts podem se comunicar com outros hosts na mesma VLAN, eliminando o problema de hosts enviando mensagens de Broadcast para todos os outros dispositivos da rede e consumindo a largura de banda necessária.

Mas e se um host em uma VLAN precisar se comunicar com um host em uma VLAN diferente? Se você for um administrador de rede, você sabe que as pessoas vão querer se comunicar com outras pessoas fora da sua rede. É aqui que o roteamento entre VLAN pode ajudá-lo.

Arquitetura de Rede: O roteamento entre VLAN usa um dispositivo de Camada 3, como um roteador ou um switch de Camada 3 (Multilayer Switch). Vamos pegar sua experiência em VLAN e combiná-la com suas habilidades de camada de rede para colocá-las à prova!

4.1.1 O que é roteamento entre VLANs?

As VLANs são usadas para segmentar redes de Camada 2 comutadas por uma variedade de razões. Independentemente do motivo, os hosts em uma VLAN não podem se comunicar com hosts em outra VLAN, a menos que haja um roteador ou um switch de Camada 3 para fornecer serviços de roteamento.

O roteamento entre VLANs (Inter-VLAN Routing) é o processo de encaminhamento de tráfego de rede de uma VLAN para outra VLAN.

Existem três opções de roteamento entre VLANs:

  • • Roteamento Inter-VLAN herdado (Legacy): Esta é uma solução legada. Não escala bem.
  • • Router-on-a-stick: Esta é uma solução aceitável para uma rede de pequeno a médio porte.
  • • Switch de Camada 3 usando SVIs (Switched Virtual Interfaces): Esta é a solução mais escalável para organizações de médio a grande porte.

4.1.2 Roteamento entre VLANs legadas

A primeira solução de roteamento entre VLANs dependia do uso de um roteador com várias interfaces Ethernet físicas. Cada interface do roteador era conectada a uma porta de switch em VLANs diferentes. As interfaces do roteador serviram como Default Gateways para os hosts locais na sub-rede de cada VLAN.

Por exemplo, consulte a topologia abaixo em que o roteador R1 possui duas interfaces físicas conectadas ao switch S1.

A tabela de endereços MAC (MAC Address Table) do switch S1 é preenchida dinamicamente da seguinte forma com base no mapeamento físico:

  • A porta Fa0/1 está associada à VLAN 10 e conecta-se à interface G0/0/0 de R1.
  • A porta Fa0/11 está associada à VLAN 10 e conecta-se ao PC1.
  • A porta Fa0/12 está associada à VLAN 20 e conecta-se à interface G0/0/1 de R1.
  • A porta Fa0/24 está associada à VLAN 20 e conecta-se ao PC2.

Quando o PC1 envia um pacote para o PC2 em outra rede, ele o encaminha para o seu Default Gateway 192.168.10.1. O roteador R1 recebe o pacote em sua interface física G0/0/0, examina o endereço IP de destino do pacote e realiza o roteamento, encaminhando o pacote para fora de sua interface física G0/0/1 em direção à porta F0/12 (pertencente à VLAN 20) no switch S1. Por fim, o switch S1 encaminha o quadro para o PC2.

Limitação Crítica de Escabilidade: O roteamento entre VLANs legado funciona, mas possui uma limitação severa: não é escalável. Como os roteadores possuem um número restrito de interfaces físicas, a exigência de uma interface física exclusiva por VLAN esgota rapidamente a capacidade de hardware do equipamento.

Neste cenário, se houvesse seis ou mais VLANs separadas para interconexão, seriam necessárias seis interfaces físicas distintas no roteador para atender ao tráfego.

Observação Histórica: Este método legado de roteamento baseado em interfaces físicas dedicadas caiu em desuso nas redes comutadas modernas, sendo abordado neste material estritamente para fins didáticos e conceituais.

4.1.3 Roteamento de router-on-a-stick entre VLANs

O método de roteamento entre VLANs “router-on-a-stick” supera a limitação do modelo legado. Ele requer apenas uma interface Ethernet física para rotear o tráfego entre várias VLANs distintas em uma rede.

Uma interface Ethernet do roteador Cisco IOS é configurada como um link de Trunk 802.1Q e conectada a uma porta em modo Trunk em um switch de Camada 2. Especificamente, a interface do roteador é dividida logicamente usando Subinterfaces para identificar as VLANs roteáveis.

As Subinterfaces são interfaces virtuais baseadas em software, onde cada uma está associada a uma única interface física. Elas são configuradas independentemente com um endereço IP e encapsulamento de VLAN correspondente à sua sub-rede, facilitando o roteamento lógico.

Como demonstrado na simulação interativa, quando o PC1 na VLAN 10 se comunica com o PC3 na VLAN 30, o fluxo ocorre da seguinte forma:

  • R1 recebe o tráfego Unicast devidamente marcado (tag 802.1Q) pela VLAN 10 por meio de sua interface física de entrada.
  • O roteador direciona internamente o pacote para a subinterface lógica correspondente, analisa o endereço IP de destino e toma a decisão de roteamento de Camada 3.
  • Os dados são encapsulados novamente com a nova tag da VLAN 30 e devolvidos pela mesma interface física em direção ao switch.
  • O switch S1 (ou S2 conforme a malha de distribuição) remove a marca (untag) da VLAN do quadro e entrega o tráfego limpo ao host de destino na porta de acesso F0/23.

Limitação de Engenharia de Redes: O método Router-on-a-stick é recomendado e perfeitamente aceitável para redes de pequeno e médio porte, porém ele não escala de forma eficiente ao ultrapassar a marca de aproximadamente 50 VLANs devido à contenção e disputa de largura de banda na mesma interface física comum.

4.1.4 Roteamento Inter-VLAN em um Switch de Camada 3

O método moderno e de alta performance para a execução de roteamento entre VLANs consiste no uso de Switches de Camada 3 através de SVIs (Switched Virtual Interfaces).

Definição de Hardware: Um switch de Camada 3 também é amplamente conhecido como Multilayer Switch (Switch Multicamada), pois possui a capacidade nativa de operar tanto na Camada 2 (comutação por endereço MAC) quanto na Camada 3 (roteamento por endereço IP).

As SVIs criadas para o roteamento entre redes são configuradas exatamente da mesma maneira que a tradicional interface de VLAN de gerência. Embora seja uma estrutura totalmente virtualizada em software, a SVI desempenha rigorosamente as mesmas funções de gateway que uma interface física de roteador executaria, provendo o processamento e encaminhamento de pacotes para todas as portas lógicas associadas àquela respectiva VLAN.

Vantagens da Arquitetura Multicamada

  • Altíssima Performance: São substancialmente mais velozes que o modelo Router-on-a-stick, pois todo o tráfego é comutado e roteado diretamente por hardware via circuitos integrados específicos (ASICs).
  • Otimização de Infraestrutura: Elimina por completo a necessidade de cabos ou links físicos externos interligando o switch central ao roteador para realizar o tráfego inter-VLAN.
  • Escalabilidade de Banda: Não ficam limitados às restrições de uma única interface comum, possibilitando a agregação de links de Trunk através de agrupamentos EtherChannel na Camada 2 para expandir a largura de banda.
  • Latência Reduzida: A latência de transmissão é drasticamente inferior, visto que os pacotes de dados não precisam deixar o chassi do switch principal para serem roteados em sub-redes distintas.
  • Padrão de Mercado: Constituem a topologia de escolha mais comum para a implementação do núcleo (Core/Distribution) de redes locais corporativas e redes de campus (LAN).

Fator de Custo: A principal contrapartida no mapeamento desse design de redes reside exclusivamente no custo financeiro dos equipamentos, uma vez que Switches de Camada 3 robustos apresentam um valor de aquisição mais elevado comparados aos switches convencionais.
