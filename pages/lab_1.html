1.2 Scripts de Configuração do Switch (S1)

Comandos aplicados via terminal de console para segurança básica, criptografia e isolamento na VLAN de gerência:

Switch> enable
Switch# configure terminal

hl=pt-br
! Definição do Nome e Segurança Global
hostname S1
no ip domain-lookup
service password-encryption
enable secret class

! Mensagem de Banner MOTD
banner motd #Unauthorized access is strictly prohibited.#

! Configuração de Segurança de Linha
line con 0
 password cisco
 login
 logging synchronous
exit

line vty 0 15
 password cisco
 login
exit

! Criação e Atribuição da VLAN 99 de Gerência
interface vlan 99
 ip address 192.168.1.2 255.255.255.0
 ipv6 address 2001:db8:acad:1::2/64
 ipv6 address fe80::2 link-local
 no shutdown
exit

! Mapeamento da Interface Física e Gateway
interface GigabitEthernet1/0/6
 switchport access vlan 99
exit
ip default-gateway 192.168.1.1

! Salvando na NVRAM
exit
copy running-config startup-config

1.3 Configuração Crítica do Host (PC-A)

Para o correto funcionamento do laboratório e o sucesso dos testes de ping e telnet, a placa de rede do PC-A deve ser alterada de DHCP para Static.

Atenção: Deixar a placa em modo DHCP causará falha crítica de comunicação (100% de perda de pacotes) devido à ausência de um servidor DHCP ativo na topologia física local.

1.4 Respostas e Perguntas para Reflexão

1. Por que você deve usar uma conexão de console para configurar inicialmente o switch?

Resposta: Porque um switch de fábrica vem completamente limpo, sem endereçamento IP ou configurações de rede ativas. O acesso via console permite o gerenciamento direto fora de banda (out-of-band).

2. Por que o comando 'login' é necessário nas linhas de acesso?

Resposta: Ele ativa explicitamente a autenticação na linha escolhida, forçando o sistema do Cisco IOS a solicitar e checar a senha configurada quando uma tentativa de conexão ocorrer.

3. Por que mudar a VLAN 1 padrão para um número de VLAN diferente (ex: VLAN 99)?

Resposta: Segurança estrutural. Como a VLAN 1 é associada nativamente a todas as portas por padrão, movendo os IPs de gerência para a VLAN 99 isolamos o tráfego administrativo do tráfego comum dos usuários, prevenindo ataques como o VLAN hopping.

4. Como você pode impedir que credenciais e senhas sejam enviadas em texto limpo pela rede?

Resposta: Localmente no switch, usamos o comando service password-encryption. Para o tráfego remoto nas linhas VTY, substituímos o uso do protocolo Telnet pelo protocolo seguro e criptografado SSH (Secure Shell).
