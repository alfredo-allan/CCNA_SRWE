2.0 Escopo de Execução da CLI (Linha por Linha)

2.1 Switch Central SWA (Core / Distribuição)

Responsável por concentrar e criar a base de dados de VLANs, amarrar o entroncamento estático seguro com o SWB e negociar dinamicamente o link com o SWC via DTP. 

SWA> enable
! Entra no modo de privilégio administrativo (Modo EXEC Privilegiado).

SWA# configure terminal
! Abre o modo de configuração global do switch.

SWA(config)# vlan 10
SWA(config-vlan)# name Admin
SWA(config-vlan)# vlan 20
SWA(config-vlan)# name Contas
SWA(config-vlan)# vlan 30
SWA(config-vlan)# name HR
SWA(config-vlan)# vlan 40
SWA(config-vlan)# name Voz
SWA(config-vlan)# vlan 99
SWA(config-vlan)# name Gerenciamento
SWA(config-vlan)# vlan 100
SWA(config-vlan)# name Nativa
SWA(config-vlan)# exit
! Cria as VLANs lógicas e define os nomes estritos exigidos pelo validador.

SWA(config)# interface GigabitEthernet0/1
! Entra na interface conectada diretamente ao switch SWB.

SWA(config-if)# switchport mode trunk
! Força a porta a operar permanentemente como um link de Tronco (Trunk).

SWA(config-if)# switchport trunk native vlan 100
! Altera a VLAN nativa do padrão (VLAN 1) para a VLAN 100, evitando falhas de segurança.

SWA(config-if)# switchport nonegotiate
! Desliga o envio de frames DTP, impedindo que a porta aceite negociações externas.
SWA(config-if)# exit

SWA(config)# interface GigabitEthernet0/2
! Entra na interface conectada diretamente ao switch SWC.

SWA(config-if)# switchport mode dynamic desirable
! Define a porta para negociar ativamente com o SWC, forçando a subida do tronco automático.

SWA(config-if)# switchport trunk native vlan 100
! Alinha a VLAN nativa 100 para que corresponda a toda a malha de troncos.
SWA(config-if)# exit

SWA(config)# interface vlan 99
! Cria a Switched Virtual Interface (SVI) para administração.

SWA(config-if)# ip address 192.168.99.252 255.255.255.0
! Atribui o endereço IP de gerência e máscara de sub-rede do SWA.

SWA(config-if)# no shutdown
! Ativa eletricamente a interface virtual.
SWA(config-if)# end

SWA# write memory
! Salva permanentemente todas as configurações na NVRAM do aparelho.

2.2 Switch SWB (Acesso / Hosts 1, 2 e 3)

Mapeia os computadores locais diretamente em suas respectivas VLANs e fixa o entroncamento estático seguro na porta de uplink. 

SWB> enable
SWB# configure terminal

! [Criação idêntica de VLANs 10, 20, 30, 40, 99 e 100 com seus respectivos nomes]

SWB(config)# interface FastEthernet0/1
SWB(config-if)# switchport mode access
SWB(config-if)# switchport access vlan 10
! Configura a porta do PC1 como modo de acesso e a vincula exclusivamente à VLAN 10 (Admin).

SWB(config)# interface FastEthernet0/2
SWB(config-if)# switchport mode access
SWB(config-if)# switchport access vlan 20
! Configura a porta do PC2 como modo de acesso e a vincula à VLAN 20 (Contas).

SWB(config)# interface FastEthernet0/3
SWB(config-if)# switchport mode access
SWB(config-if)# switchport access vlan 30
! Configura a porta do PC3 como modo de acesso e a vincula à VLAN 30 (HR).
SWB(config-if)# exit

SWB(config)# interface GigabitEthernet0/1
! Entra na interface de uplink que vai para o SWA.

SWB(config-if)# switchport mode trunk
SWB(config-if)# switchport trunk native vlan 100
SWB(config-if)# switchport nonegotiate
! Replica as mesmas configurações de Trunk fixo e Hardening aplicadas na outra ponta do cabo no SWA.
SWB(config-if)# exit

SWB(config)# interface vlan 99
SWB(config-if)# ip address 192.168.99.253 255.255.255.0
SWB(config-if)# no shutdown
SWB(config-if)# end
SWB# write memory

2.3 Switch SWC (Acesso / Hosts 4, 5, 6 e Telefone IP)

Executa a inserção dos computadores de dados, ativa o encapsulamento simultâneo de voz com QoS na porta F0/4 e responde ao DTP dinâmico do Core. 

SWC> enable
SWC# configure terminal

! [Criação idêntica de VLANs 10, 20, 30, 40, 99 e 100 com seus respectivos nomes]

SWC(config)# interface FastEthernet0/1
SWC(config-if)# switchport mode access
SWC(config-if)# switchport access vlan 10

SWC(config)# interface FastEthernet0/2
SWC(config-if)# switchport mode access
SWC(config-if)# switchport access vlan 20

SWC(config)# interface FastEthernet0/4
! Entra na porta física onde estão conectados o Telefone IP e o PC de dados simultaneamente.

SWC(config-if)# switchport mode access
SWC(config-if)# switchport access vlan 10
! Determina a VLAN 10 (Admin) para processar os pacotes normais de dados da máquina.

SWC(config-if)# switchport voice vlan 40
! Ativa a marcação de QoS, separando os frames de áudio do telefone para a VLAN 40 (Voz).
SWC(config-if)# exit

SWC(config)# interface GigabitEthernet0/2
! Entra na interface conectada ao cabo vindo da porta G0/2 do SWA.

! Nota: Esta interface permanece com o DTP padrão ("dynamic auto"). Ao receber as 
! solicitações da ponta "dynamic desirable" do SWA, elas fecham o Trunk de forma automatizada.
SWC(config-if)# switchport trunk native vlan 100
! Alinha a VLAN nativa do link dinâmico para a VLAN 100.
SWC(config-if)# exit

SWC(config)# interface vlan 99
SWC(config-if)# ip address 192.168.99.254 255.255.255.0
SWC(config-if)# no shutdown
SWC(config-if)# end
SWC# write memory

3.0 Glossário de Engenharia de Redes (Explicado para Leigos)

Se um switch de fábrica funciona como um grande salão de festas aberto onde todos se escutam ao mesmo tempo, as configurações aplicadas criam divisórias acústicas e corredores privados inteligentes:

Conceito Técnico Analogia e Significado Prático
VLAN (Virtual LAN) Dividir um switch físico em vários switches digitais isolados. É colocar paredes de drywall no salão de festas para que uma reunião de negócios não escute a música do aniversário ao lado.
Link de Tronco (Trunk) Uma super-estrada digital em um único cabo que consegue carregar o tráfego de todas as divisórias (VLANs) ao mesmo tempo entre switches, sem misturar os dados.
DTP (Dynamic Trunking Protocol) O protocolo de conversação que permite a duas portas de switches decidirem sozinhas, via cabo, se devem virar uma rua comum (Acesso) ou uma super-estrada (Trunk).
Dynamic Desirable O modo "proativo" de negociação. A porta do switch fica ativamente chamando o switch vizinho do outro lado do cabo, dizendo: "Ei, vamos virar um Trunk agora?".
Switchport Nonegotiate Uma trava rígida de segurança. Você ordena para a porta do switch: "Você é um Trunk fixo, não converse com ninguém de fora e ignore qualquer tentativa de negociação".
VLAN de Voz (Voice VLAN) Uma faixa de ambulância exclusiva na avenida. Ela separa o tráfego dos Telefones IP (telefones de mesa ligados à internet) para que a voz passe com prioridade e nunca trave ou fique robotizada.
SVI (Switched Virtual Interface) Uma porta digital invisível onde colocamos um endereço IP dentro do switch, permitindo que o administrador acesse o painel de controle do aparelho à distância, sem precisar ir até o rack físico.

4.0 Aplicação e Impacto no Mercado Corporativo

Caso 1: Proteção e Isolamento de Setores (VLANs Admin, Contas, HR)

Em uma empresa, os computadores da recepção (VLAN 10), do financeiro (VLAN 20) e do RH (VLAN 30) compartilham o mesmo switch físico. Sem a configuração de VLANs, se uma secretária abrir um anexo com vírus na recepção, o malware se espalharia instantaneamente pelos computadores da diretoria e do financeiro. Com as VLANs estritas, os setores operam em bolhas de segurança isoladas: se um setor for comprometido, os outros continuam intactos.

Caso 2: Convergência de Telefonia Corporativa Sem Quedas (Voice VLAN 40)

Na mesa de um funcionário corporativo, chega apenas um cabo de rede da parede que se conecta ao Telefone IP, e do telefone sai outro cabo para o computador. Sem a VLAN de Voz, se o funcionário iniciar o download de um arquivo pesado no computador, o tráfego de internet esmagaria os pacotes de áudio, fazendo a chamada cair. A Voice VLAN cria uma fila prioritária no switch para o telefone, garantindo ligações estáveis mesmo sob uso severo de banda.

Caso 3: Hardening e Proteção contra Espionagem Industrial (Nonegotiate)

Links de tronco padrão que usam negociação dinâmica (DTP) ativa abrem uma brecha crítica de segurança. Um atacante físico poderia desconectar o cabo de um computador comum e plugar uma máquina modificada para simular um switch corporativo. O switch real seria enganado pelo DTP e abriria uma super-estrada (Trunk), vazando os dados de todos os setores (ataque conhecido como VLAN Hopping). O comando switchport nonegotiate blinda o canal contra isso.

Caso 4: Provisionamento Ágil de Novos Blocos (Dynamic Desirable)

Ao expandir a infraestrutura para um novo galpão ou andar de estoque, um switch novo é instalado de fábrica (SWC) com as portas em modo automático padrão. Em vez de o engenheiro se deslocar fisicamente até o local isolado para configurar as portas de interconexão, o switch principal (SWA) toma a iniciativa através do modo dynamic desirable. Assim que o cabo é conectado, o switch core propõe o fechamento do tronco e a nova estrutura é integrada de forma automatizada.
