2.0 Ativação de Trunking e Tagging no Switch S1

Configuração dos canais de distribuição para permitir que múltiplas VLANs compartilhem o mesmo meio físico através de tags 802.1Q: 

S1> enable
S1# configure terminal
S1(config)# interface GigabitEthernet0/1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport trunk native vlan 99

S1(config)# interface GigabitEthernet0/2
S1(config-if)# switchport mode trunk
S1(config-if)# switchport trunk native vlan 99
S1(config-if)# end
S1# write memory

3.0 Resolução do Erro Native VLAN Mismatch (S2 & S3)

O protocolo Cisco Discovery Protocol (CDP) acusará inconsistência de tráfego sem tag enquanto as duas pontas do enlace não utilizarem o mesmo ID nativo.

Análise de Syslog: O erro de mismatch bloqueia temporariamente o tráfego de controle na VLAN nativa. Aplique a padronização para a VLAN 99 nos switches periféricos para cessar os alertas. 
! --- PARAMETRIZAÇÃO DO SWITCH S2 ---
S2(config)# interface GigabitEthernet0/1
S2(config-if)# switchport mode trunk
S2(config-if)# switchport trunk native vlan 99

! --- PARAMETRIZAÇÃO DO SWITCH S3 ---
S3(config)# interface GigabitEthernet0/2
S3(config-if)# switchport mode trunk
S3(config-if)# switchport trunk native vlan 99

4.0 Auditoria de Estados dos Troncos

Comandos essenciais para validar se o entroncamento foi fechado com sucesso e verificar quais IDs lógicos estão autorizados a transitar:

S1# show interface gigabitEthernet 0/1 switchport
Validação Concluída: A saída correta exibirá o modo como on ou desirable (DTP) e o encapsulamento definido estritamente como 802.1q.

3.5.1 Introdução ao DTP

Alguns switches Cisco têm um protocolo proprietário que permite negociar automaticamente o entroncamento com um dispositivo vizinho. Esse protocolo é chamado DTP (Dynamic Trunking Protocol). O DTP pode acelerar o processo de configuração de um administrador de rede.

As interfaces de tronco Ethernet suportam diferentes modos de entroncamento. Uma interface pode ser definida como entroncamento ou não-entroncamento, ou para negociar entroncamento com a interface do vizinho. A negociação de tronco é gerenciada pelo DTP, que opera somente ponto a ponto, entre dispositivos de rede.

O DTP é ativado automaticamente nos switches Cisco Catalyst das séries 2960 e 3560. Ele gerencia a negociação de tronco apenas se a porta do switch vizinho estiver configurada em um modo de tronco com suporte para DTP. Os switches de outros fornecedores não oferecem suporte a esse protocolo.

Cuidado (Caution): Alguns dispositivos de internetworking podem encaminhar quadros DTP incorretamente, o que pode causar configurações incorretas. Para evitar isso, desative o DTP nas interfaces de switch Cisco conectadas a dispositivos que não suportam o protocolo.

Desativando o DTP (Nonegotiate)

A configuração DTP padrão para switches Cisco Catalyst 2960 e 3560 é o modo dynamic auto.

Para habilitar o entroncamento de um switch Cisco para um dispositivo que não suporta DTP, utilize o comando switchport mode trunk em conjunto com o comando de configuração de interface switchport nonegotiate. Isso faz com que a interface se torne um tronco fixo, mas interrompe a geração e o envio de quadros DTP: 

S1(config)# interface fa0/1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport nonegotiate

Reativando o Protocolo de Entroncamento Dinâmico

Para restabelecer a negociação automática e reativar o protocolo de entroncamento dinâmico (DTP), remova a diretiva de não-negociação limpando a linha de comando da interface: 

S1(config-if)# no switchport nonegotiate

Isso permitirá que a interface volte a enviar e processar os quadros DTP para sincronizar o estado do link automaticamente com o switch remoto.

3.5.2 Modos de interface negociados

O comando switchport mode possui opções adicionais para negociar dinamicamente o modo operacional de Camada 2 (acesso ou tronco) da interface física com o switch vizinho. A sintaxe de configuração completa do comando é a seguinte:

Switch(config-if)# switchport mode { access | dynamic { auto | desirable } | trunk }

Modos Operacionais e Comportamentos do DTP

Abaixo estão descritas as quatro opções disponíveis e o comportamento de hardware correspondente para cada uma delas:

Modo de Configuração Comportamento Operacional da Interface
access Coloca a interface no modo de acesso permanente não-tronco. A interface negocia para se converter em um link de acesso comum, mesmo que o vizinho queira entroncar.
dynamic auto Torna a interface capaz de converter o link em um tronco se a interface vizinha estiver configurada para o modo tronco ou desejável (desirable). Este é o padrão em switches mais novos. A porta apenas escuta, ela não inicia a negociação ativamente.
dynamic desirable Faz com que a interface tente ativamente converter o link em um tronco. A porta envia quadros DTP de forma ativa e solicita à interface vizinha que mude para o modo tronco.
trunk Coloca a interface no modo de entroncamento permanente. Mesmo que a porta vizinha não concorde, a interface força o modo de tronco e continua a enviar quadros DTP periódicos.

Interrompendo a Negociação Atômica do DTP

Use o comando de configuração de interface switchport nonegotiate para interromper de forma definitiva a negociação DTP. Sob essa diretiva, o switch para de gerar frames de sincronização dinâmica nesta interface. 

S1(config)# interface fa0/1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport nonegotiate
Regras de Aplicação Estritas: Você pode usar o comando switchport nonegotiate somente quando o modo de porta de comutação (switchport mode) da interface for explicitamente definido como access ou trunk. Tentativas de aplicar a não-negociação em portas configuradas como dynamic auto ou dynamic desirable resultarão em erro de sintaxe. Você deve configurar manualmente a interface vizinha como tronco para estabelecer a conectividade.

3.5.3 Resultados de uma configuração de DTP

A tabela abaixo ilustra os resultados combinatórios das opções de configuração do protocolo DTP quando definidas nas extremidades opostas de um link local interconectando duas portas de switches Cisco Catalyst 2960.

Como diretriz de arquitetura de redes, a melhor prática é configurar links de tronco estaticamente sempre que possível e desabilitar o DTP para mitigar vulnerabilidades de segurança e falhas de negociação.

Matriz de Sincronização Lógica do Link

Cruze a configuração do Switch Local com o modo do Switch Remoto para determinar o estado operacional resultante da linha de transmissão:

Switch Local \ Remoto Dynamic Auto Dynamic Desirable Trunk Access
Dynamic Auto Access Trunk Trunk Access
Dynamic Desirable Trunk Trunk Trunk Access
Trunk Trunk Trunk Trunk Incompatível (Erro)
Access Access Access Incompatível (Erro) Access

Entendendo a Lógica da Matriz

A negociação bem-sucedida de um tronco depende do envio de requisições ou da concordância explícita mútua:

  • Dynamic Auto + Dynamic Auto = Access: Como ambas as pontas estão configuradas no modo passivo (apenas escutam e esperam uma solicitação), nenhuma delas toma a iniciativa de enviar um quadro de controle DTP requisitando o entroncamento. O link se consolida como acesso padrão.
  • Dynamic Desirable + Dynamic Auto = Trunk: O modo desejável toma a iniciativa de enviar ativamente pacotes DTP. Ao receber a proposta, a porta em modo auto aceita a transição, estabelecendo o tronco de forma dinâmica.
  • Trunk + Access = Erro Lógico: Enquanto um lado força o encapsulamento de tags 802.1Q e aguarda frames multiplexados, o outro ignora os cabeçalhos extras e trata o canal de forma plana. Isso resulta em tráfego descartado e inconsistência operacional na topologia.

3.5.4 Verificando o modo de DTP

O modo DTP padrão de uma interface depende diretamente da versão do Cisco IOS Software e da plataforma de hardware utilizada no comutador. Para determinar o estado operacional e o modo de negociação atual do protocolo, execute o comando de diagnóstico show dtp interface interface-id.

Saída do Comando de Diagnóstico DTP no Switch S1

Abaixo está detalhado o mapeamento de variáveis internas da máquina de estados do protocolo obtido via CLI:

S1# show dtp interface fa0/1

TOS/TAS/TNS: ACCESS/AUTO/ACCESS
TOT/TAT/TNT: NATIVE/NEGOTIATE/NATIVE
Neighbor address 1: C80084AEF101
Neighbor address 2: 000000000000
Hello timer expiration (sec/state): 11/RUNNING
Access timer expiration (sec/state): never/STOPPED
Negotiation timer expiration (sec/state): never/STOPPED
Multidrop timer expiration (sec/state): never/STOPPED
FSM state: S2:ACCESS
# times multi & trunk 0
Enabled: yes
In STP: no

Entendendo as Siglas da Saída (TOS/TAS/TNS)

A linha TOS/TAS/TNS e a linha TOT/TAT/TNT resumem o status lógico da interface física:

  • TOS (Type Operational Status): Exibe o estado operacional atual da porta de rede (neste caso, operando como ACCESS).
  • TAS (Type Administrative Status): Exibe o modo administrativo configurado na interface (neste caso, AUTO, indicando dynamic auto).
  • TNS (Type Neighbor Status): Exibe o status detectado no switch vizinho conectado à outra ponta do link.
  • FSM state (Finite State Machine): Identifica a fase atual da Máquina de Estados Finitos do protocolo (neste caso, estabilizada em S2:ACCESS).

Diretrizes de Endurecimento (Hardening) e Segurança

Observação (Melhor Prática Geral): Em ambientes de produção corporativos, a recomendação de segurança padrão é configurar os links de tronco estaticamente usando os comandos switchport mode trunk e switchport nonegotiate.

Da mesma forma, em portas destinadas a computadores de usuários finais (onde o entroncamento não é desejado), a porta deve ser explicitamente travada com o comando switchport mode access para desativar o DTP e blindar o switch contra ataques de infiltração por salto de VLAN (VLAN Hopping).
