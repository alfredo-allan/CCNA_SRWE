Laboratório Prático - DHCPv6

Configurar DHCPv6

Implementação de SLAAC, DHCPv6 Stateless e DHCPv6 Stateful em topologia IPv6 com roteadores Cisco.

1.0 Diferenças entre SLAAC, DHCPv6 Stateless e DHCPv6 Stateful

Antes de iniciar a configuração prática, é fundamental entender os três métodos de atribuição dinâmica de endereços IPv6 que serão utilizados neste laboratório.

1. SLAAC (Stateless Address Autoconfiguration)

O que é? O próprio host (PC) gera seu endereço IPv6 automaticamente, sem precisar de servidor DHCP.

Como funciona?

  • O roteador envia RA (Router Advertisement) com o prefixo da rede
  • O host combina o prefixo com seu endereço MAC (EUI-64) ou gera um endereço aleatório
  • O host cria um endereço IPv6 completo
✅ Endereço IPv6: Gerado pelo host ❌ DNS Server: Não fornecido ❌ Nome de domínio: Não fornecido ✅ Gateway: Obtido via RA 🟡 Controle do admin: Baixo 🏷️ Flag RA: Nenhuma

2. DHCPv6 Stateless (Sem Estado)

O que é? O host gerencia seu próprio endereço (via SLAAC), mas usa o DHCP APENAS para informações adicionais (DNS, domínio, etc.).

Como funciona?

  • O roteador envia RA com a flag other-config-flag (flag O)
  • O host gera seu endereço via SLAAC
  • O host consulta o servidor DHCP apenas para obter DNS e domínio
  • O DHCP NÃO atribui o endereço IPv6
✅ Endereço IPv6: Gerado pelo host ✅ DNS Server: DHCP fornece ✅ Nome de domínio: DHCP fornece ✅ Gateway: Obtido via RA 🟡 Controle do admin: Médio 🏷️ Flag RA: other-config-flag (O)

3. DHCPv6 Stateful (Com Estado)

O que é? O servidor DHCP controla tudo — atribui o endereço IPv6, DNS, domínio e outras configurações.

Como funciona?

  • O roteador envia RA com a flag managed-config-flag (flag M)
  • O host ignora o SLAAC e solicita tudo do DHCP
  • O servidor DHCP atribui o endereço IPv6 completo
  • O servidor DHCP também fornece DNS, domínio, etc.
✅ Endereço IPv6: DHCP atribui ✅ DNS Server: DHCP fornece ✅ Nome de domínio: DHCP fornece ✅ Gateway: RA ou DHCP 🟢 Controle do admin: Total 🏷️ Flag RA: managed-config-flag (M)

1.4 Verificar SLAAC no PC-A

Configuração do PC-A: Configure a placa de rede para "Obter um endereço IPv6 automaticamente" e "Obter o endereço do servidor DNS automaticamente".

Verificar configuração

C:\> ipconfig

IPv6 Address. . . . . . . . . . . : 2001:db8:acad:1:5c43:ee7c:2959:da68
Link-local IPv6 Address . . . . .: fe80::5c43:ee7c:2959:da68%6
Gateway Padrão . . . . . . . . .: fe80::1%6
Pergunta: De onde veio a parte do host-id do endereço?

Resposta: A parte do host-id (últimos 64 bits) foi gerada automaticamente pelo PC usando o método EUI-64 ou endereço temporário aleatório, baseado no endereço MAC da placa de rede.

1.5 Configurar DHCPv6 Stateless no R1

Instrução de Validação: Lembre-se de configurar a flag other-config-flag na interface G0/0/1.

Criar pool DHCP Stateless

R1# configure terminal
R1(config)# ipv6 dhcp pool R1-STATELESS
R1(config-dhcp)# dns-server 2001:db8:acad::254
R1(config-dhcp)# domain-name Stateless.com
R1(config-dhcp)# exit

Aplicar na interface G0/0/1

R1(config)# interface g0/0/1
R1(config-if)# ipv6 nd other-config-flag
R1(config-if)# ipv6 dhcp server R1-STATELESS
R1(config-if)# end
R1# write memory

Verificar configuração

R1# show ipv6 dhcp pool
R1# show ipv6 dhcp server
Após reiniciar o PC-A, a saída do ipconfig /all deve mostrar: 
Sufixo de DNS primário. . . . . . . : Stateless.com
Específico de Conexão Sufixo DNS. : STATELESS.com
Lista de pesquisa de sufixo DNS. . : Stateless.com
Servidores DNS . . . . . . . . . . . : 2001:db8:acad::254

1.6 Configurar DHCPv6 Stateful no R1

Instrução de Validação: O pool Stateful deve conter o prefixo 2001:db8:acad:3::/64 para atender o PC-B.

Criar pool DHCP Stateful

R1# configure terminal
R1(config)# ipv6 dhcp pool R2-STATEFUL
R1(config-dhcp)# address prefix 2001:db8:acad:3::/64
R1(config-dhcp)# dns-server 2001:db8:acad::254
R1(config-dhcp)# domain-name StateFul.com
R1(config-dhcp)# exit

Aplicar na interface G0/0/0

R1(config)# interface g0/0/0
R1(config-if)# ipv6 dhcp server R2-STATEFUL
R1(config-if)# end
R1# write memory

1.7 Configurar Relay DHCPv6 no R2

ATENÇÃO: O comando ipv6 dhcp relay destination pode não estar disponível em algumas versões do IOS (como a imagem universalk9). Se o comando não for reconhecido, utilize a SOLUÇÃO ALTERNATIVA abaixo.

Configurar Relay (se disponível)

R2# configure terminal
R2(config)# interface g0/0/1
R2(config-if)# ipv6 nd managed-config-flag
R2(config-if)# ipv6 dhcp relay destination 2001:db8:acad:2::1
R2(config-if)# end
R2# write memory
SOLUÇÃO ALTERNATIVA (Caso o relay não funcione):

Configure o R2 como servidor DHCP diretamente para a rede do PC-B:

R2# configure terminal
R2(config)# ipv6 dhcp pool R2-STATEFUL
R2(config-dhcp)# address prefix 2001:db8:acad:3::/64
R2(config-dhcp)# dns-server 2001:db8:acad::254
R2(config-dhcp)# domain-name StateFul.com
R2(config-dhcp)# exit
R2(config)# interface g0/0/1
R2(config-if)# ipv6 nd managed-config-flag
R2(config-if)# ipv6 dhcp server R2-STATEFUL
R2(config-if)# end
R2# write memory
