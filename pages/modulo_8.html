SLAAC e DHCPv6

Endereçamento dinâmico IPv6 para redes modernas — automatize a atribuição de endereços e simplifique a gestão de infraestrutura.

8.0.1 Bem-vindo

Bem-vindo ao SLAAC e DHCPv6!

SLAAC e DHCPv6 são protocolos de endereçamento dinâmico para uma rede IPv6. Então, um pouco de configuração tornará o seu dia como administrador de rede muito mais fácil. Neste módulo, você aprenderá como usar o SLAAC para permitir que os hosts criem seu próprio endereço Unicast Global IPv6, bem como configurar um roteador Cisco IOS para ser um servidor DHCPv6, um cliente DHCPv6 ou um agente de retransmissão DHCPv6. Este módulo inclui um laboratório onde você configurará o DHCPv6 em equipamentos reais!

R1 (DHCPv6 Server)
SW1 (Relay Agent)
PC1 (DHCPv6 Client)
Visão Geral do Módulo: Este módulo aborda os três principais métodos de atribuição de endereços IPv6: SLAAC (Stateless Address Autoconfiguration), DHCPv6 Stateless e DHCPv6 Stateful. Você aprenderá as diferenças entre eles e quando aplicar cada um.

SLAAC

Autoconfiguração sem estado — hosts geram seu próprio endereço IPv6 usando EUI-64 ou Privacy Extensions.

DHCPv6 Stateless

Fornece informações adicionais como DNS, NTP e Domain Name, mas o endereço IPv6 vem do SLAAC.

DHCPv6 Stateful

Atribuição completa de endereços IPv6, similar ao DHCPv4, com concessão e renovação de endereços.

Preparando o Roteador para IPv6

Antes de configurar o SLAAC ou DHCPv6, certifique-se de que o roteador tenha IPv6 habilitado:

R1#configure terminal
R1(config)#ipv6 unicast-routing
% IPv6 routing enabled
R1(config)#interface gigabitethernet 0/0/0
R1(config-if)#ipv6 address 2001:db8:acad:1::1/64
R1(config-if)#ipv6 enable
R1(config-if)#no shutdown
%LINK-5-CHANGED: Interface GigabitEthernet0/0/0, changed state to up
R1(config-if)#end
R1#show ipv6 interface brief

GigabitEthernet0/0/0    [up/up]
    FE80::1
    2001:DB8:ACAD:1::1
Dica do Engenheiro: O comando ipv6 unicast-routing é obrigatório para habilitar o roteamento IPv6 no roteador. Sem ele, o dispositivo não processará pacotes IPv6 entre interfaces.

8.1.1 Configuração de host IPv6

Comece pelo começo

Para usar a configuração automática de endereço sem estado (SLAAC) ou DHCPv6, você deve revisar endereços Unicast Globais (GUAs) e endereços de Link Local (LLAs). Este tópico abrange ambos.

Em um roteador, um endereço Unicast Global IPv6 (GUA) é configurado manualmente usando o comando ipv6 address ipv6-address prefix-length no modo de configuração de interface.

Um host Windows também pode ser configurado manualmente com uma configuração de endereço IPv6 GUA, conforme mostrado na figura abaixo.

Configuração Manual de IPv6 no Windows

Inserir manualmente um GUA IPv6 pode ser demorado e suscetível a erros. Por isso, a maioria dos hosts Windows estão habilitados para adquirir dinamicamente uma configuração IPv6 GUA.

Preferred DNS server: 2001:db8:acad:1::1

Configuração Manual: O endereço 2001:db8:acad:1::10/64 é um exemplo de GUA configurado manualmente em um host Windows, com o gateway padrão apontando para 2001:db8:acad:1::1 (geralmente o roteador local).

Configuração Automática de IPv6

A maioria dos hosts Windows estão habilitados para adquirir dinamicamente uma configuração IPv6 GUA, conforme mostrado na figura abaixo.

Internet Protocol Version 6 (TCP/IPv6) Properties
─ ☐ ✕

IPv6 address: (empty)

Configuração Automática: Com esta opção selecionada, o host utilizará SLAAC ou DHCPv6 para obter seu endereço IPv6, prefixo de rede, gateway padrão e servidores DNS automaticamente.

Configuração de GUA em um Roteador Cisco

Para configurar um endereço IPv6 GUA em um roteador Cisco IOS, utilize o comando ipv6 address no modo de configuração de interface: 

R1#configure terminal
R1(config)#interface gigabitethernet 0/0/0
R1(config-if)#ipv6 address 2001:db8:acad:1::1/64
R1(config-if)#ipv6 enable
R1(config-if)#no shutdown
%LINK-5-CHANGED: Interface GigabitEthernet0/0/0, changed state to up
R1(config-if)#end
R1#show ipv6 interface brief

GigabitEthernet0/0/0    [up/up]
    FE80::1
    2001:DB8:ACAD:1::1

R1#show ipv6 route

IPv6 Routing Table - default - 5 entries
Codes: C - Connected, L - Local, S - Static, ...
C   2001:DB8:ACAD:1::/64 [0/0]
     via GigabitEthernet0/0/0, directly connected
L   2001:DB8:ACAD:1::1/128 [0/0]
     via GigabitEthernet0/0/0, local
Observação: O comando ipv6 enable ativa automaticamente o IPv6 na interface e gera um endereço Link Local (LLA) baseado no endereço MAC da interface (EUI-64). O LLA é usado para comunicação na rede local e é essencial para o funcionamento do SLAAC e DHCPv6.

Métodos de Configuração IPv6

Método Configuração Vantagem Desvantagem
Manual Administrador define o endereço Controle total sobre o endereçamento Propenso a erros, trabalhoso em redes grandes
SLAAC Host gera seu próprio endereço Automatizado, sem necessidade de servidor Não fornece DNS ou outras configurações
DHCPv6 Stateless SLAAC + informações adicionais do DHCP Combina autoconfiguração com informações de rede Requere servidor DHCPv6 para informações extras
DHCPv6 Stateful Servidor DHCP atribui endereço completo Controle centralizado, auditoria de endereços Requere servidor DHCPv6, mais complexo

8.1.2 Endereço local do link do host IPv6

Endereço de Link Local IPv6

Quando o endereçamento IPv6 automático é selecionado, o host tentará obter e configurar automaticamente as informações de endereço IPv6 na interface. O host usará um dos três métodos definidos pela mensagem de Internet Control Message Protocol versão 6 (ICMPv6) Router Advertisement (RA) recebida na interface.

Um roteador IPv6 que está no mesmo link que o host envia mensagens RA que sugerem aos hosts como obter suas informações de endereçamento IPv6. O endereço de Link Local IPv6 é criado automaticamente pelo host quando ele é inicializado e a interface Ethernet está ativa.

Verificando o Endereço Link Local

A saída do comando ipconfig abaixo mostra um endereço de Link Local (LLA) gerado automaticamente em uma interface Windows. 

C:\PC1>ipconfig

Windows IP Configuration

Ethernet adapter Ethernet0:

   Connection-specific DNS Suffix  . :
   IPv6 Address. . . . . . . . . . . :
   Link-Local IPv6 Address . . . . . . : fe80::fb:1d54:839f:f595%21
   IPv4 Address. . . . . . . . . . . : 169.254.202.140
   Subnet Mask . . . . . . . . . . . : 255.255.0.0
   Default Gateway . . . . . . . . . :

C:\PC1>
Observação Importante: Observe que a interface não criou um GUA IPv6. A razão é porque, neste exemplo, o segmento de rede não tem um roteador para fornecer instruções de configuração de rede para o host.
ID de Zona (Scope ID): Os sistemas operacionais do host mostrarão às vezes um endereço de link local anexado com um "%" e um número (ex: %21). Isso é conhecido como ID de zona ou ID de escopo. Ele é usado pelo SO para associar o LLA a uma interface específica.
RFC 3315: DHCPv6 é definido na RFC 3315 — Dynamic Host Configuration Protocol for IPv6 (DHCPv6).

Métodos de Configuração via Router Advertisement (RA)

O host usará um dos três métodos definidos pela mensagem ICMPv6 Router Advertisement (RA) recebida na interface:

SLAAC

O host gera seu próprio GUA usando o prefixo recebido no RA e o identificador EUI-64 ou Privacy Extensions.

RA Flag: A=1, O=0

DHCPv6 Stateless

O host usa SLAAC para o GUA, mas utiliza DHCPv6 para obter informações adicionais (DNS, NTP, Domain Name).

RA Flag: A=1, O=1

DHCPv6 Stateful

O host obtém todo o endereçamento (GUA, DNS, gateway) via DHCPv6, similar ao DHCPv4.

RA Flag: A=0, M=1

Como o Host Gera o Endereço Link Local

O endereço Link Local IPv6 é criado automaticamente pelo host quando ele é inicializado e a interface Ethernet está ativa. O processo segue estas etapas:

1 O host identifica o endereço MAC da interface Ethernet
2 Insere FF:FE no meio do endereço MAC (processo EUI-64)
3 Inverte o 7º bit (Universal/Local bit) do primeiro byte
4 Prefixa com FE80::/10 para formar o endereço Link Local 
Exemplo de geração do LLA a partir do MAC:

MAC Address:    00-1A-2B-3C-4D-5E
   ↓ Inserir FF:FE
EUI-64:         001A:2BFF:FE3C:4D5E
   ↓ Inverter 7º bit (00 → 02)
EUI-64 Mod:     021A:2BFF:FE3C:4D5E
   ↓ Prefixo FE80::/10
Link Local:     FE80::21A:2BFF:FE3C:4D5E

Tipos de Endereços IPv6

Tipo Escopo Descrição Exemplo
Unicast Global (GUA) Global (Internet) Endereço público roteável na Internet 2001:db8:acad:1::1
Link Local (LLA) Link Local (mesmo segmento) Comunicação na rede local, não roteável FE80::21A:2BFF:FE3C:4D5E
Unique Local (ULA) Local (site) Endereço privado, similar ao IPv4 RFC 1918 FD00::/8
Multicast Variável Comunicação one-to-many FF02::1 (todos os nós)
Anycast Variável Comunicação one-to-nearest 2001:db8::1

8.1.3 Atribuição de GUA IPv6

Design Simplificado do IPv6

O IPv6 foi projetado para simplificar como um host pode adquirir sua configuração IPv6. Por padrão, um roteador habilitado para IPv6 anuncia suas informações IPv6. Isso permite que um host crie ou adquira dinamicamente sua configuração IPv6.

O GUA IPv6 pode ser atribuído dinamicamente usando serviços sem estado e com estado, conforme mostrado na figura abaixo.

Todos os métodos sem estado e com estado neste módulo usam mensagens de RA ICMPv6 para sugerir ao host como criar ou adquirir sua configuração IPv6. Embora os sistemas operacionais host sigam a sugestão do RA, a decisão real depende, em última análise, do host.

Métodos de Atribuição Dinâmica do GUA

Somente SLAAC

Sem Estado
  • O roteador envia mensagens de Router Advertisement (RA), fornecendo todas as informações de endereço IPv6 (prefixo da rede, comprimento do prefixo e gateway padrão)
  • Os hosts usam as informações de RA exclusivamente para todos os endereços, incluindo a criação de seu próprio GUA
Host → RS Roteador → RA (Prefixo, Gateway) Host → Gera GUA

SLAAC com DHCPv6

Sem Estado
  • As mensagens de RA do roteador fornecem informações de configuração IPv6 aos hosts e informam que contatem um servidor DHCPv6 sem estado para obter informações adicionais
  • Os hosts usam as informações de RA para criar seu próprio GUA exclusivo e obter informações adicionais de um servidor DHCPv6
Host → RS Roteador → RA (A=1, O=1) Host → DHCPv6 (Info Extra)

DHCPv6 Stateful

Com Estado
  • As mensagens de RA do roteador informam os hosts para contatar um servidor DHCPv6 com estado ou roteador habilitado para DHCPv6 para obter todas as informações de configuração IPv6, exceto o endereço de gateway padrão
  • Os hosts contatam um servidor DHCPv6 para adquirir todas as informações de endereço IPv6
  • O host obtém informações de gateway padrão das mensagens de RA do roteador
Host → RS Roteador → RA (A=0, M=1) Host → DHCPv6 (GUA, DNS)

Comparação dos Métodos de Atribuição

Método Tipo Flags RA GUA Informações Extras Gateway
Somente SLAAC Sem Estado A=1, O=0 Host gera via EUI-64 Não fornece Do RA
SLAAC + DHCPv6 Stateless Sem Estado A=1, O=1 Host gera via EUI-64 DNS, NTP, Domain via DHCPv6 Do RA
DHCPv6 Stateful Com Estado A=0, M=1 DHCPv6 atribui DNS, NTP, Domain via DHCPv6 Do RA

Configurando as Flags RA no Roteador

As flags RA são configuradas no roteador para indicar ao host qual método de atribuição utilizar:

R1#configure terminal

// SLAAC Only (A=1, O=0)
R1(config)#interface gigabitethernet 0/0/0
R1(config-if)#ipv6 nd prefix 2001:db8:acad:1::/64
R1(config-if)#ipv6 nd managed-config-flag
R1(config-if)#ipv6 nd other-config-flag

// SLAAC + DHCPv6 Stateless (A=1, O=1)
R1(config-if)#ipv6 nd other-config-flag
R1(config-if)#ipv6 dhcp server NOME_POOL
// DHCPv6 Stateful (A=0, M=1)
R1(config-if)#ipv6 nd managed-config-flag
R1(config-if)#ipv6 dhcp server NOME_POOL

R1(config-if)#end
R1#show ipv6 interface gigabitethernet 0/0/0

Dica do Engenheiro:
  • ipv6 nd managed-config-flag → Define a flag M (Managed) = 1 → DHCPv6 Stateful
  • ipv6 nd other-config-flag → Define a flag O (Other) = 1 → DHCPv6 Stateless
  • Se ambas as flags estiverem desabilitadas → SLAAC Only

Entendendo as Flags RA

A Autonomous Flag

A=1 → Host pode usar SLAAC para criar GUA
A=0 → Host NÃO pode usar SLAAC

O Other Config Flag

O=1 → Host deve usar DHCPv6 para outras configurações (DNS, NTP, Domain)

M Managed Config Flag

M=1 → Host deve usar DHCPv6 Stateful para obter GUA e outras configurações
